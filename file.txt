  1. A user visits a URL → Flask routes it in app.py

  3. All pages extend base.html (shared navbar, styles, etc.)
  4. database/db.py handles reading/writing to the SQLite database

❯    /btw ist fast api and flask same                                           

● No, they're different:

  ┌────────────────┬──────────────────────────┬───────────────────────────┐
  │                │          Flask           │          FastAPI          │
  ├────────────────┼──────────────────────────┼───────────────────────────┤
  │ Speed          │ Slower                   │ Much faster               │
  ├────────────────┼──────────────────────────┼───────────────────────────┤
  │ Type hints     │ Optional                 │ Required                  │
  ├────────────────┼──────────────────────────┼───────────────────────────┤
  │ Auto docs      │ No                       │ Yes (Swagger UI built-in) │
  ├────────────────┼──────────────────────────┼───────────────────────────┤
  │ Best for       │ Web apps with HTML pages │ APIs / backend services   │
  ├────────────────┼──────────────────────────┼───────────────────────────┤
  │ Learning curve │ Easier                   │ Slightly steeper          │
  └────────────────┴──────────────────────────┴───────────────────────────┘

  This project uses Flask because it's better suited for rendering HTML pages
  (like login, register, expense pages).

  FastAPI is better when you're building a pure API (e.g., a backend for a
  React/mobile app).
